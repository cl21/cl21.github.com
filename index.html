CL21

CL21 is an experimental project redesigning Common Lisp.

Features

  • More object oriented.
  • Add more functional programming facilities.
  • Organize symbols into several packages.
  • Include MOP.
  • Syntax for regular expression.
  • Written in pure Common Lisp.

Motivation

Common Lisp has succeeded.

— Common Lisp the Language, 2nd Edition

Dear Common Lispers,

Common Lisp is the standardized most powerful language with high-performance implementations in the world. It has lexical closures, multi-dispatch object system called CLOS, the metaobject protocol and macros.

On the other hand, its community does not get a lot of success (at least in Japan). Most young Lispers are hacking with Ruby or Clojure.

Why? Common Lisp is much faster than them. Ruby has no macros and reader macros. Then, why?

Actually, they don't require them. Because their language is already designed well enough for most people. They don't mind its speed either.

Is Common Lisp well-designed enough? I don't think so. There're similar functions for different data types (elt, aref and nth). Long name for macros commonly used (destructuring-bind and multiple-value-bind). The order of arguments does not have consistency (getf and gethash). Its learning cost is very high.

In such condition, can non-Common Lispers believe that Common Lisp is the most expressive one?

Fortunately, we can change the interface much better by abstracting the implementation with functions, macros and reader macros. If you believe our language is the most expressive, let's justify it.

We should consider about the future of the language for the next generation, not only for us.

Written on January 26, 2014 by Eitarow Fukamachi.
